Interactive 3D Visualization with COVISE

COVISE is an open-source software environment for integration of simulation, postprocessing and visualization. It is designed for visualization of scientific data, collaborative working and visualization in VR/AR environments through dedicated rendering modules. This allows visualization on Desktops as well as CAVEs, HMDs, or Powerwalls.

COVISE uses a modular structure to separate between different processing steps. This enables distribution of steps across several machines, moreover, enabling processing on HPC platforms.
Besides the modular structure allows flexible assembling of processing workflows. These can be assemble using COVISE's graphical user interface.

construction Installation

COVISE is supported on various platforms. COVISE builds can be downloaded from:
https://fs.hlrs.de/projects/covise/support/download/
Besides, COVISE can be obtained from the following sources:

  • Windows
    On Windows, an Installer is available here:
    https://fs.hlrs.de/projects/covise/support/download/snapshot-2021-12-17/

  • MacOS
    On MacOS, COVISE can be installed through Homebrew:

    brew install hlrs-vis/tap/covise

  • UNIX For UNIX systems, building from source code is enabled by acquiring the code from GitHub:
    https://github.com/hlrs-vis/covise
    There also build requirements are listed. Once, these are ensured, COVISE can be installed:

    git clone https://github.com/hlrs-vis/covise.git --recursive
    cd covise
    source .covise.sh
    make

bolt Quick Start

COVISE can be invoked from the terminal by:

covise

or, on Windows, started through the Application Icon This will start the COVISE Mapeditor (Figure 1) which is composed into several areas: a workflow area, showing the assembled map; a module browser to add modules to the map; a message area where Modules can print messages and a menu tool bar.

Figure 1: The COVISE GUI

A workflow can be assembled by dragging modules from the browser to the workflow area.
Modules a divided into categories:

  • IO: Import datasets from files
  • Mapper: Map data to geometries (e.g. iso surfaces)
  • map data to color values
  • Tracer: trace particles and compute streamlines
  • Filter: extract subsets of larger datasets (e.g. cutting surfaces)
  • Tools: apply further tools, such as cutting off selected parts
  • Renderer: display the results

Modules are represented by a blue rectangle. On top and bottom of it, input and output ports are shown as squares. These can be interconnected between different modules to pass data from one to another. However, it is required to pass matching data types. The book icon on the modules opens the module parameter settings where additional adjustments can be made, such as to set the path to a data file or define an iso value. For some modules, these settings can also be interactively changed from within the renderer. An individual module can be executed by double-click on it. All subsequent modules will execute as well. The full map can be executed by clicking on the cogwheel in the menu toolbar.
Once a renderer is added to the workflow, the rendering window will open-up in a separate window (Figure 2). This provides additional menus and allows for interaction and navigation using the mouse.

Figure 2: The OpenCOVER window, showing the rendering

domain_verification Examples

Examples of COVISE (.net) files contain a preconfigured processing map of modules, dedicated to a selected use case. Several examples come with the installation of COVISE. They can be loaded into COVISE by clicking on the folder icon in the menu tool bar. This should open the covise/net directory where a subdirectory for examples is located. Select a .net file by double-click and a preconfigured module map will be loaded.
